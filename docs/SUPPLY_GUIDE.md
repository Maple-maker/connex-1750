# SUPPLY GUIDE — BOM Ingest, Box Assignment, and Individual Items

Audience: supply sergeant or armorer responsible for uploading BOM PDFs, assigning equipment to boxes, and handling edge cases (zero-on-hand, individual items, exclusions).

---

## What is a BOM?

A BOM (Bill of Material) represents one end item — for example, one M4A1 Carbine with its serial number — plus all of its component line items. The tool reads BOM PDFs and creates one BOM card per end item.

BOM PDFs are generated by the property book system or unit supply. Each filename encodes the item: model, LIN, serial number, and (optionally) bumper number.

---

## Where BOM ingest happens

BOM ingest is part of **Step 3 (PACKING)** — the 2D split-screen. The left panel contains the BOM pool and the individual item form. The right panel contains the box cards.

---

## Step 1 — Uploading BOM PDFs

1. Click **Upload BOM PDFs** in the left panel, or drag and drop files onto the upload zone.
2. Drop all BOM PDFs for this connex at once — the tool parses them in bulk.
3. Each BOM becomes a card in the left tray, showing:
   - **Nomenclature** — the item name (e.g. RIFLE M4A1)
   - **LIN** — Line Item Number
   - **NSN** — National Stock Number (sniffed from the PDF if present)
   - **Serial** — serial number from the filename

The parser is shape-based and order-independent — it classifies each filename token by its format (LIN = 6-char alpha-numeric, serial = long alphanumeric string, etc.) rather than by position. Cards with missing LIN or model are flagged with an amber warning.

**What if a BOM card shows missing data?** Click the card to edit the LIN or nomenclature before assigning.

---

## Step 2 — Assigning BOMs to boxes

Each BOM card in the left panel must be assigned to exactly one box in the right panel.

**Drag and drop:**
1. Drag a BOM card from the left panel.
2. Drop it onto a box card in the right panel. The target box highlights on hover.

**Click to assign:**
1. Click a BOM card to select it (card highlights gold).
2. Click the target box card. The BOM is assigned and appears on the box card.

**Assignment moves (also available via the API for advanced use):**

| Move type | What it does |
|-----------|-------------|
| `{bom_id, box_num}` | Assign BOM to a specific box number |
| `{bom_id, separate: true}` | Pull this BOM into its own dedicated box (next free box number) |
| `{bom_id, exclude: true}` | Remove this BOM from the packing list entirely |

Use **separate** for sensitive items or items that need their own DD1750. Use **exclude** for items that are not shipping in this connex (deadlined, retained, etc.).

---

## Step 3 — Filling SLOC and SHRH POC

After assigning BOMs, fill the required fields on each box card:

- **SLOC** — Storage Location Code. Where is this box physically located? (e.g. `BLDG-100`, `YARD-A3`, `ISO PAD 2`). Required before sealing.
- **SHRH POC** — Sub-Hand Receipt Holder, Point of Contact. Who is accountable for the items in this box? Enter the name in RANK LAST, FIRST format (e.g. `CPT JONES`). Required before sealing.

**Box status badge:**

| Badge | Meaning |
|-------|---------|
| Empty (gray) | No BOMs or individual items assigned |
| Needs SLOC/SHRH (amber) | Has content but a required field is missing |
| Complete (green) | Has content, SLOC filled, SHRH POC filled — ready to seal |

All boxes must show **Complete** (green) before the connex can be sealed.

---

## Adding individual items

Individual items are loose items not covered by BOM PDFs — tools, accessories, non-standard components, or items too few to have their own BOM. Add them in the **Add Individual Item** form at the bottom of the left panel in Step 3 — there is no separate step for this.

1. Select which box the item belongs to.
2. Fill any combination of:
   - **Description** — free text (e.g. "SLING ASSY, SINGLE POINT")
   - **SN** — serial number
   - **NSN** — 13-digit National Stock Number
   - **LIN** — 6-character Line Item Number
3. All fields are optional. An item with only a description is valid.
4. Click **Add Item**. The item appears on the box card.

Individual items appear on the DD1750 for that box and in the SITREP individual item count.

> **Fast-follow note:** A planned AI helper (owl-alpha) will suggest NSN and LIN values from a description. The hook is in the form but not yet connected.

## Zero-on-hand handling

A zero-on-hand condition occurs when a box's BOM items all have a quantity of zero — equipment that appears on the property book but is not physically present.

The tool handles zero-on-hand as follows:

- Zero-quantity items are tracked and rendered on the DD1750 with a `0` quantity.
- The SITREP flags any box with all-zero quantities.
- Zero-on-hand is **not** a seal blocker. The connex can be sealed with zero-on-hand boxes as long as SLOC and SHRH POC are filled.
- The S4 must reconcile zero-on-hand items against the property book before certifying the packing list.

**Why does the tool not block sealing on zero-on-hand?** Because zero-on-hand is a property book discrepancy to be resolved, not a packing error. The PDF documents the condition; the SITREP flags it for the S4.

---

## What the downloaded ZIP contains

After sealing at Step 5 (Review & Seal):

| File | Contents |
|------|---------|
| `Master_1750.pdf` | All boxes condensed onto a single master DD1750, organized by box number |
| `Box_001.pdf`, `Box_002.pdf`, ... | One DD1750 per occupied box with all BOM rows, individual items, SLOC, SHRH POC, and battalion stamp |

Empty boxes are not included in the ZIP.

---

## Re-generating after changes

If you need to update an individual item or add a late BOM after sealing:

1. Go back to Step 3 (click the step dot in the stepper) and make your changes.
2. Re-seal at Step 4/5 and download a new ZIP.

Alternatively, the API supports direct updates: `PUT /api/connex/<connex_id>` to update box individual_items, then `POST /api/connex/<connex_id>/generate` to get a new ZIP.

## Troubleshooting ingest

**BOM card shows no LIN or model:** The filename could not be parsed. Edit the card manually in the tray or rename the file to include the LIN (format: `A12345`) and re-upload.

**Ingest returns no cards:** The PDF may be image-only (scanned). The tool attempts OCR if tesseract is installed; otherwise the PDF is skipped. Check the server logs for `OCR_AVAILABLE` status.

**Cards disappear after server restart:** BOM ingest state is in-memory. Re-upload BOM PDFs after any restart. Connex and profile JSON files are persisted to disk and will be there when the server comes back up.

**Need fewer boxes:** You cannot reduce the box count after the connex is created without starting a new connex. Exclude BOMs from extra boxes (`{bom_id, exclude: true}`) — empty boxes are skipped in the PDF output. Or create a new connex in Step 2 with the correct count.
