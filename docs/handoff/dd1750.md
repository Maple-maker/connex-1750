## Render function signatures

### `render_core.build_connex_header(...) -> HeaderInfo`
```python
from render_core import build_connex_header, draw_master_header

header = build_connex_header(
    connex: dict,       # full Connex dict (from connex_store)
    box: dict,          # one entry from connex["boxes"]
    box_count: int,     # total number of boxes in this connex
    box_nums_label: str, # range-compressed label, e.g. "1-5" (pass "" if single box)
    profile: dict = None, # optional Profile dict for stamp_text/battalion fields
) -> HeaderInfo
```
Returns a `HeaderInfo` populated with all connex fields. Blank `sun`/`connex_no`/`seal_no` are automatically replaced with `[SUN PENDING]`/`[CONNEX PENDING]`/`[SEAL PENDING]`.

### Per-box PDF generation (existing API, now connex-aware)
```python
from render_core import generate_dd1750_from_items, draw_master_header
from master_core import rows_to_bom_items

items = rows_to_bom_items(box_rows)   # box_rows = list of row dicts for this box
path, count = generate_dd1750_from_items(
    items=items,
    template_path="blank_1750.pdf",
    output_path="/tmp/box1.pdf",
    header=header,                        # from build_connex_header()
    draw_master_header_fn=draw_master_header,
)
# Returns (output_path: str, item_count: int)
# Battalion stamp is drawn automatically when header.stamp_text is non-empty.
```

### `sitrep_render.render_sitrep_pdf(sitrep_dict, output_path=None) -> bytes | str`
```python
from sitrep_render import render_sitrep_pdf

# Write to file:
path = render_sitrep_pdf(sitrep_dict, output_path="/tmp/sitrep.pdf")

# Get bytes (for streaming response):
raw_bytes = render_sitrep_pdf(sitrep_dict)
```
`sitrep_dict` must conform to Contract C (§5). Missing/blank connex fields display as they appear in the dict — Backend should pass the placeholder strings through from the Connex model rather than expecting sitrep_render to fill them.

## HeaderInfo extension (new fields)
`HeaderInfo` now has six additional optional fields (all default to `""`):
| Field | Purpose |
|-------|---------|
| `sloc` | Storage Location code |
| `shrh_poc` | SHRH Property Officer name |
| `sun` | System Unit Number (use `[SUN PENDING]` if blank) |
| `connex_no` | Physical container number (use `[CONNEX PENDING]` if blank) |
| `seal_no` | Seal number (use `[SEAL PENDING]` if blank) |
| `stamp_text` | Battalion stamp text; empty = no stamp rendered |

All existing callers (legacy 2D flow) pass no values for these fields and are unaffected — defaults are `""`, which suppresses the new rendering paths.

## Layout caveats
- **PACKED BY cell** can now hold up to 6 lines (was 4). The cell is tall enough; tested against the real blank_1750.pdf coordinates.
- **Stamp position**: lower-right signature area (x≈478, y≈72), rotated 12° CCW. If the signer name is very long it may visually abut the stamp — Backend should keep `signed_by` under 30 chars.
- **SITREP pagination**: ReportLab `KeepTogether` keeps each box block on one page where possible; very long BOM lists (50+ rows) will still page-break mid-box. This is intentional and correct — data is never truncated.
- **Weight tracking** is explicitly out of scope; no weight fields exist anywhere in this module.

## Known gaps / TODO for downstream
- Backend must call `build_connex_header` + `rows_to_bom_items` + `generate_dd1750_from_items` per box to assemble the ZIP in `POST /api/connex/<id>/generate`. Backend owns ZIP assembly and box-row derivation from the Connex model.
- `sitrep_render` expects `seal_no` key on connex sub-dicts (Contract C uses `seal_no`). Backend's `sitrep.py` must match this key.
- The stamp is text-only (stencil style). No image stamp support — out of scope for MVP.
